《花花妙屋》合成页面玩法优化策划案

（基于业界竞品调研 + 项目现状分析，9项优化功能详细设计，可直接交付开发）

一、调研背景与目标

1. 调研目的

针对《花花妙屋》合成经营主页面（棋盘合成区域），调研业界合成类小游戏的合成页面玩法特点，对标分析我们的优势与不足，提出具体优化方案。

2. 优化目标

• 保底目标：合成页面体验不低于四季合合、浪漫餐厅等业界标杆
• 突破目标：在业界标配功能基础上，通过花店主题差异化创新，形成独有竞争优势
• 核心方向：体验增强 + 操作效率 + 情感差异化

3. 调研范围

• 四季合合（四季物语微信小游戏版）—— 核心对标
• 浪漫餐厅 —— 同品类竞品
• 庄园合合（Merge Mansion 国服）—— 品类标杆
• EverMerge —— 海外头部参考

二、业界竞品合成页面分析

1. 四季合合（四季物语微信小游戏版）

定位：国内合成类小游戏标杆，Merge-2品类代表

合成页面核心机制：

（1）合成方式：二合一（Merge-2）
• 两个相同物品拖拽合成，100%成功，无冷却
• 合成反馈：闪光粒子+音效，约0.5秒，不阻断操作

（2）生成器系统
• 产出源道具（带闪电标志+计时器），点击消耗体力产出随机物品
• 分永久型（CD冷却，冷却结束后可再次点击）和消耗型（用完消失）
• 广告可加速CD重置

（3）棋盘迷雾探索
• 初始少量格子开放，合成波及解锁周围迷雾格
• 含可窥视格（半透明可见物品，可跨格合成）
• 含钥匙格（金币解锁）
• 解锁动画有爽感

（4）宝箱合成
• 宝箱可合成升级（铜→银→金），高级宝箱产出更好
• 点击宝箱消耗体力

（5）体力系统
• 点击生成器/宝箱消耗体力，3分钟恢复1点
• 体力为0时不可点击生成器和宝箱

（6）订单系统
• 客人头顶气泡显示需求，完成订单获得金币+建材

（7）合成线提示（重要UX功能）
• 点击物品右上角感叹号可浏览物品完整合成线
• 展示当前物品在合成链中的位置，前后级别物品一目了然
• 帮助新手快速理解"这个东西能合成什么"

（8）广告加速
• 看广告跳过生成器CD、获取气泡物品

2. 浪漫餐厅

定位：同品类竞品，在四季合合基础上做了部分差异化

合成页面差异化机制：

（1）蜘蛛网格子
• 锁定格内有物品被"蜘蛛网"缠绕，合成解锁释放空格
• 比四季合合的迷雾格更具视觉趣味和互动感
• 蜘蛛网需要在旁边合成特定物品才能清除

（2）仓库系统（核心差异）
• 棋盘外有独立仓库空间，可暂存不急用的物品
• 仓库容量初始4格，可用宝石扩容至8~12格
• 核心空间管理手段：棋盘满时将暂时不需要的物品拖入仓库，腾出合成空间
• 仓库物品可随时拖回棋盘

（3）母棋机制
• 高级生成器称为"母棋"，产出特定品类的物品
• 母棋需要通过合成升级获得

（4）冰冻棋子（惊喜机制）
• 特殊触发出现带倒计时的冰冻高级棋子
• 在倒计时内与同级棋子合成可跳级（直接升2级）
• 增加策略惊喜感和紧迫感

（5）格子扩容
• 宝石购买扩容 + 合成解锁，双重途径
• 给玩家持续扩展棋盘的目标感

3. 庄园合合（Merge Mansion 国服）

定位：品类标杆，全球头部合成类游戏

合成页面特色：

（1）竖版棋盘+二合一
• 开创了竖版Merge-2品类分支，竖屏操作更适合手机

（2）1000+可合成物品
• 超大合成图鉴，合成链极其丰富
• 长线内容深度远超同品类

（3）产出源道具视觉区分
• 带闪电标志的是产出源（生成器），不带的是合成道具
• 视觉区分非常清晰，新手一看就懂

（4）任务驱动合成
• 合成特定物品推进故事剧情，解锁庄园区域
• 合成有明确目标感，而非盲目合成

（5）空间管理核心乐趣
• 有限棋盘格子内做好资源管理是核心玩法
• 何时清理低级物品、何时保留材料，是持续决策点

（6）合成线可视化
• 点击物品感叹号查看完整合成路径
• 路径以图文形式展示，当前物品高亮标记

（7）沙漏道具
• 消耗型道具，使用后加速生成器CD
• 作为宝箱奖励或任务奖励获得

4. EverMerge

定位：海外头部合成游戏，与Merge-2有品类差异

合成页面差异：

（1）三合一（Merge-3）
• 3个相同物品合成为1个高级物品
• 5个同时合成可产出2个高级物品（鼓励积攒）
• 策略维度不同：攒更多再合成效率更高

（2）大地图开放世界
• 非棋盘格，自由放置在大地图上
• 地图可滚动、缩放，空间压力较小

（3）奇幻主题
• 合成出龙、狮鹫等奇幻生物
• 主题差异化明显

2. 差距总结

（1）明显缺失（业界标配，我们没有）：
• 合成线可视化 —— 四季合合、庄园合合均有，是玩家理解玩法的关键入口
• 仓库/临时存储 —— 浪漫餐厅的核心差异化功能，解决棋盘空间紧张的核心痛点

（2）需要补全（策划有提但未完整落地）：
• 空闲可合成提示（物品呼吸发光）—— 策划案十七章关键交互细节第9条有提及，需确认代码实现完整度

（3）可创新突破（业界普遍没有，我们可以做出差异）：
• 连锁合成连击系统 —— 增加操作爽感
• 双击快速合成 —— 提升操作效率
• 花语合成彩蛋 —— 花店主题独有的情感差异化
• 智能合成引导线 —— 降低新手决策负担
• 季节主题棋盘 —— 长线新鲜感
• 合成统计面板 —— 成就感和数据驱动留存

四、优化方案一：合成线可视化（优先级 P0）

1. 功能概述

玩家点击棋盘上任意物品后，可查看该物品所在合成链的完整路径，了解当前物品可以合成什么、需要什么材料、还差几步到顶级。

2. 需求来源

• 四季合合：点击物品右上角感叹号，弹出合成链浮层
• 庄园合合：点击物品感叹号，查看完整合成路径
• 两款标杆产品均有此功能，是玩家理解玩法和制定合成策略的核心入口

3. 详细玩法描述

（1）入口方式
• 方式一：玩家点击选中棋盘上某个物品后，底部 ItemInfoBar 中出现「合成线」按钮（小书本图标）
• 方式二：长按棋盘上某个物品超过0.8秒，直接弹出合成线浮层
• 两种入口并存，覆盖不同操作习惯

（2）合成线浮层内容
• 浮层从屏幕底部上滑弹出，高度约占屏幕40%，半透明遮罩背景
• 浮层标题：显示当前物品所属品类和花系/饮品线名称（如"浪漫花系合成线"、"茶饮线合成线"、"花束建筑合成线"）
• 合成链展示：水平排列展示该物品所在完整合成链的所有等级物品

（3）展示规则
• 每个等级显示：物品图标（彩色）+ 物品名称 + 等级标签
• 当前选中物品：金色高亮边框 + 底部三角标记"当前"
• 已拥有的物品等级（棋盘上有同类）：图标正常彩色显示
• 未拥有的物品等级：图标略灰（降低饱和度30%），但仍可辨认
• 顶级物品（满级）：右上角显示皇冠小图标
• 建筑材料合成线额外显示：合成到顶级后转化为哪个建筑（箭头连接建筑图标）
• 等级之间用箭头"→"连接，箭头下方显示"×2"表示需要2个合成

（4）附加信息
• 浮层底部显示："合成 2个 [当前物品名] 可获得 [下一级物品名]"
• 如果是满级物品，显示："已达到最高等级！"
• 宝箱合成线独立展示：铜宝箱 → 银宝箱 → 金宝箱

4. 交互流程

步骤1：玩家点击/长按棋盘上某个物品
步骤2：
  - 若点击：底部 ItemInfoBar 显示物品信息 + 出现「合成线」按钮
  - 若长按（超过0.8秒）：直接弹出合成线浮层
步骤3：玩家点击「合成线」按钮（若走点击路径）
步骤4：合成线浮层从底部上滑弹出（0.3秒动画）
步骤5：玩家可左右滑动查看完整合成链（若链路长度超出屏幕宽度）
步骤6：玩家点击合成链中任一物品图标，可查看该物品的详细信息（名称、说明文本）
步骤7：点击浮层外部区域或浮层右上角关闭按钮，关闭浮层

5. 数值设计

• 无数值消耗，纯信息展示功能
• 涉及的合成链数据：
  ◦ 花束：3花系 × 6级 = 3条合成链
  ◦ 花饮：3饮品线 × 3级 = 3条合成链
  ◦ 建筑材料：花束建筑线4级 + 饮品建筑线3级 = 2条合成链
  ◦ 宝箱：1条合成链（3级）
  ◦ 总计：9条合成链

6. UI设计要点

• 浮层背景：深色半透明（rgba(0,0,0,0.7)），内容区域为圆角白色/米色卡片
• 合成链水平排列，单个物品卡片宽度约60px，间距20px
• 箭头使用花瓣形状装饰箭头（契合花店主题），而非生硬的直线箭头
• 当前物品高亮效果：金色呼吸光效边框（1秒循环）
• 字体：物品名称12px，等级标签10px加粗
• 关闭按钮：右上角圆形×按钮
• 支持左右滑动手势浏览长链路

7. 与现有系统的关联

• 数据来源：复用 ItemConfig.ts 中的 ITEM_DEFS、FLOWER_DATA、DRINK_DATA、BUILDING_MAT_DATA
• 合成关系：复用 getMergeResultId() 函数的逻辑
• UI入口：集成到现有 ItemInfoBar 组件中，新增「合成线」按钮
• 长按入口：在 BoardView 的物品交互逻辑中增加长按检测（现有为拖拽）

8. 开发量预估

• 前端UI：合成线浮层组件（1天）
• 数据处理：从 ItemConfig 生成合成链数据结构（0.5天）
• 交互集成：ItemInfoBar按钮 + 长按检测（0.5天）
• 总计：1~2天

五、优化方案二：仓库系统（优先级 P0）

1. 功能概述

在棋盘外提供独立的暂存空间（仓库），玩家可将棋盘上暂时不需要的物品拖入仓库暂存，需要时再拖回棋盘。解决棋盘空间紧张的核心痛点。

2. 需求来源

• 浪漫餐厅：棋盘外有独立仓库，可用宝石扩容，是核心空间管理手段
• 庄园合合：有类似的"泡泡"暂存机制，物品可以暂存在棋盘外的泡泡中

3. 详细玩法描述

（1）仓库基本规则
• 仓库位于棋盘区域左下角，以小型背包/花篮图标展示
• 初始容量：4格
• 最大容量：8格（通过消耗宝石/钻石扩容）
• 仓库内物品不参与自动锁定匹配（客人订单不扫描仓库）
• 仓库内物品不可直接在仓库中合成（必须拖回棋盘才能合成）
• 仓库内物品不占用棋盘空间

（2）存入操作
• 操作方式：将棋盘上的物品拖拽到仓库图标上方，松手即存入
• 可存入的物品类型：花束、花饮、建筑材料、宝箱
• 不可存入的物品：永久型建筑（永久型建筑占据棋盘格是核心设计，不允许移除）、消耗型建筑
• 存入动画：物品缩小飞入仓库图标（0.3秒），仓库图标轻微弹跳反馈
• 仓库满时拖入：提示"仓库已满，请先取出物品"

（3）取出操作
• 操作方式一：点击仓库图标，打开仓库面板，将物品拖拽到棋盘空格上
• 操作方式二：点击仓库面板中的物品，自动放入棋盘最近的空格中
• 棋盘无空格时取出：提示"棋盘已满，请先合成或出售物品腾出空间"
• 取出动画：物品从仓库面板飞出，弹跳落入棋盘空格（0.3秒）

（4）仓库面板
• 点击仓库图标后，从底部上滑弹出仓库面板
• 面板高度约占屏幕25%
• 显示4~8个格子（根据当前容量），已存物品显示物品图标，空格显示虚线边框
• 面板顶部显示："仓库 X/Y"（当前存储量/最大容量）
• 容量未满时，右侧显示扩容按钮（"+扩容"，标注扩容价格）
• 面板外部点击或下滑关闭

（5）扩容规则
• 初始容量：4格（免费）
• 第5格：消耗50钻石
• 第6格：消耗100钻石
• 第7格：消耗200钻石
• 第8格：消耗400钻石
• 扩容为永久生效，购买一次即可

4. 交互流程

【存入物品】
步骤1：玩家长按棋盘上某个物品，物品浮起进入拖拽状态
步骤2：拖拽物品移向屏幕左下角仓库图标
步骤3：物品进入仓库图标感应区域时，仓库图标放大+高亮提示"放开存入"
步骤4：松手 → 物品缩小飞入仓库 → 棋盘格子释放为空格
步骤5：仓库图标上的数字更新（如"2/4"→"3/4"）

【取出物品】

六、优化方案三：空闲可合成提示增强（优先级 P1）

1. 功能概述

当棋盘上存在可合成的同类物品对时，通过视觉提示引导玩家发现合成机会，减少遗漏。特别在玩家静置无操作时，主动高亮引导。

2. 需求来源

• 四季合合：可合成物品有轻微闪烁提示
• 庄园合合：同类物品附近会有微弱光效引导
• 我们的策划案十七章第9条已提及"当棋盘有可合成的同品类同系同级物品时，两个物品轻微发光提示"，需确认实现完整度并增强

3. 详细玩法描述

（1）常驻微弱提示（始终生效）
• 当棋盘上存在2个或以上可合成的同品类、同系、同等级物品时：
  ◦ 这些物品的图标底部有轻微的呼吸光效（淡金色，0.5秒渐隐渐现循环）
  ◦ 光效强度较弱，不干扰正常操作，仅在仔细观察时可注意到
  ◦ 同时最多高亮2对可合成物品（避免满屏闪烁）
  ◦ 优先高亮等级最低的可合成对（引导玩家先处理低级物品释放空间）

（2）空闲主动引导（静置触发）
• 当玩家连续10秒无任何操作时：
  ◦ 棋盘上一对可合成物品的光效增强（从淡金色变为明亮金色，光效范围扩大）
  ◦ 两个物品之间出现虚线弧线连接（柔和的淡金色虚线）
  ◦ 连接线轻微闪烁2秒后恢复常驻微弱提示状态
  ◦ 如果玩家继续无操作，每15秒重复一次主动引导

（3）新手期增强
• 新手引导期间（前30分钟）：
  ◦ 常驻光效强度提升50%，更容易被注意到
  ◦ 空闲触发时间缩短为5秒（更积极引导）
  ◦ 引导时增加文字浮层提示："试试把两个相同的花合在一起吧！"（仅前3次显示文字）

（4）可关闭设置
• 设置页面增加开关："合成提示"（默认开启）
• 关闭后不再显示任何合成提示光效

4. 交互流程

【常驻提示】
系统每秒扫描一次棋盘 → 发现可合成对 → 为物品添加呼吸光效 → 物品被合成或移动后 → 重新扫描更新

【空闲引导】
玩家最后一次操作计时开始 → 10秒无操作 → 选取一对可合成物品 → 增强光效+虚线连接 → 2秒后恢复 → 继续计时 → 再15秒无操作 → 重复

5. 数值设计

| 参数 | 普通模式 | 新手模式（前30分钟） |
|------|---------|-------------------|
| 常驻光效强度 | opacity 0.3 | opacity 0.5 |
| 常驻光效节奏 | 0.5秒渐隐渐现 | 0.5秒渐隐渐现 |
| 同时高亮对数 | 最多2对 | 最多2对 |
| 空闲触发时间 | 10秒 | 5秒 |
| 引导持续时间 | 2秒 | 3秒 |
| 引导重复间隔 | 15秒 | 10秒 |
| 文字提示 | 无 | 前3次显示 |

6. UI设计要点

• 常驻呼吸光效：物品图标底部向外扩散的淡金色光晕，半径为物品图标的120%
• 空闲引导增强：光晕颜色变为明亮金色，半径扩大到150%，同时物品轻微上下浮动（2px幅度）
• 虚线连接弧线：两个物品之间的贝塞尔曲线，颜色淡金色，虚线样式（4px线段+4px间隔），线宽2px
• 光效不遮挡物品图标内容，仅作为底层装饰

7. 与现有系统的关联

• 棋盘扫描：需要基于 BoardManager 的格子数据，扫描所有已解锁格子中的物品
• 合成判断：复用现有合成条件判断逻辑（同品类+同系+同等级）
• 性能考虑：扫描频率1秒/次，棋盘最大30格，性能开销可忽略

8. 开发量预估

• 可合成对扫描逻辑（0.5天内含）
• 呼吸光效动画+空闲引导动画（0.5天内含）
• 总计：0.5天

七、优化方案四：双击快速合成（优先级 P1）

1. 功能概述

双击棋盘上的物品，自动搜索棋盘上最近的同类可合成物品并完成合成，省去拖拽操作。适合碎片化游玩场景和追求操作效率的玩家。

2. 需求来源

• 业界少有实现，属于操作创新
• 解决痛点：拖拽操作在小屏手机上不够精准，尤其棋盘格子较小时容易拖错目标

3. 详细玩法描述

（1）基本规则
• 双击棋盘上某个可合成物品（非建筑、非宝箱），系统自动执行以下逻辑：
  ① 搜索棋盘上所有同品类、同系、同等级的物品
  ② 从中找到距离当前物品最近的一个
  ③ 自动执行合成（被找到的物品飞向当前位置，合成动画播放，生成高一级物品）
• 如果棋盘上没有可合成的同类物品，双击无效果（物品轻微抖动提示"无可合成对象"）

（2）优先级规则
• 距离优先：优先合成距离最近的同类物品（曼哈顿距离）
• 锁定物品排除：已被客人订单锁定的物品不参与双击自动合成（避免意外消耗锁定物品）
• 如果最近的同类物品被锁定，则跳过该物品，寻找下一个最近的

（3）操作判定
• 双击判定：两次点击间隔 < 300ms，且两次点击位置在同一格子内
• 与拖拽的区分：拖拽需要长按50ms以上才触发拖拽状态，双击在50ms内完成两次点击
• 与单击的区分：单击选中物品（显示底部信息栏），双击为合成操作，单击后300ms内无第二次点击才判定为单击

（4）合成动画
• 被合成的远端物品沿弧线飞向当前物品位置（0.3秒）
• 到达后触发标准合成闪光特效
• 远端物品原位格子释放为空格

（5）可关闭设置
• 设置页面增加开关："双击快速合成"（默认开启）
• 关闭后双击行为等同于两次单击（选中物品）

4. 交互流程

步骤1：玩家快速双击棋盘上某个花束/花饮/建筑材料/宝箱
步骤2：系统判定为双击操作（间隔<300ms）
步骤3：搜索棋盘上最近的同类可合成物品
  - 若找到：远端物品飞向当前物品 → 合成动画 → 生成高一级物品 → 远端格子释放
  - 若未找到：当前物品轻微抖动（0.2秒）+ 浮动文字"无可合成对象"
步骤4：合成完成后，系统重新扫描可合成提示（联动优化方案三）

5. 数值设计

• 双击判定间隔：<300ms
• 拖拽触发阈值：长按>50ms + 移动>10px
• 合成飞行动画时长：0.3秒
• 抖动提示时长：0.2秒
• 无额外数值消耗，纯操作优化

6. UI设计要点

• 合成飞行轨迹：物品沿贝塞尔弧线飞向目标位置，轨迹带有淡金色拖尾粒子
• 飞行过程中物品略微缩小（0.8倍）再在目标位置恢复正常大小
• 合成完成时与拖拽合成使用相同的闪光粒子特效和音效
• "无可合成对象"浮动文字：灰色小字，在物品上方浮现并渐隐（0.8秒）

7. 与现有系统的关联

• 输入系统：需要在 BoardView 的点击事件中增加双击检测逻辑，与现有拖拽交互并存
• 合成逻辑：复用现有 MergeManager 的合成方法
• 锁定系统：双击合成需要检查 CustomerManager 的物品锁定状态

8. 开发量预估

• 双击检测逻辑（与拖拽/单击区分）（0.5天内含）
• 自动搜索最近可合成物品 + 飞行合成动画（0.5天内含）
• 总计：0.5天

八、优化方案五：连锁合成连击系统（优先级 P1）

1. 功能概述

在短时间内连续完成多次合成，触发连击计数，连击越高获得的经验加成越大。高连击触发"合成狂热"模式，带来极致操作爽感。

2. 需求来源

• 业界合成类游戏普遍缺少即时反馈强化机制
• 消除类游戏（如开心消消乐）的连击/连消机制被验证为有效的操作爽感来源
• 适用于合成类游戏，增加操作深度和技巧感

3. 详细玩法描述

（1）连击计数规则
• 每次成功合成（拖拽合成或双击合成均算），启动连击计时器
• 计时器初始时间窗口：5秒
• 在时间窗口内再次合成，连击数+1，计时器重置为5秒
• 时间窗口结束且无新合成，连击结束，显示本次连击总结

（2）连击等级与加成

| 连击数 | 连击名称 | 经验加成 | 视觉效果 |
|-------|---------|---------|---------|
| 2连 | 小连击 | +10% 经验 | 屏幕边缘淡金色光效 |
| 3连 | 连击 | +20% 经验 | 光效增强 + "3连击!"文字飘出 |
| 5连 | 大连击 | +30% 经验 | 光效再增强 + "5连击!!" 文字 + 音效升调 |
| 8连 | 超级连击 | +50% 经验 | 强光效 + "超级连击!!!" + 短暂屏幕震动 |
| 10连 | 合成狂热 | +80% 经验 | 进入合成狂热模式（详见下方） |
| 15连+ | 传说连击 | +100% 经验 | 狂热模式延续 + 特殊粒子特效 |

（3）合成狂热模式（10连击以上触发）
• 触发后持续15秒（或直到连击中断）
• 狂热期间视觉效果：
  ◦ 棋盘背景变为温暖的金色调
  ◦ 屏幕四周持续金色粒子飘落（花瓣粒子，契合花店主题）
  ◦ 所有合成特效放大1.5倍
  ◦ 背景音乐节奏加快（如果技术可实现，否则叠加快节奏音效）
• 狂热期间数值效果：
  ◦ 经验加成维持在+80%~+100%
  ◦ 合成产物有10%概率额外产出一个同等级物品（"合成溢出"，额外物品放入最近空格，无空格则不触发）

（4）连击结束结算
• 连击中断时（5秒内无新合成），屏幕中央短暂显示连击总结：
  ◦ "本次连击：X连"
  ◦ "额外经验：+XX"
  ◦ 显示1.5秒后渐隐
• 连击不影响金币、花愿、花露的获取，仅影响玩家经验值（用于升级）

（5）连击记录
• 记录玩家的"历史最高连击数"
• 在日结算（打烊结算）时展示"今日最高连击"
• 可配合优化方案九（合成统计面板）展示

4. 交互流程

步骤1：玩家完成第1次合成 → 连击计时器启动（屏幕右上角出现倒计时条，5秒）
步骤2：5秒内完成第2次合成 → 连击数变为2 → 倒计时重置 → 屏幕显示"2连击!" + 边缘光效
步骤3：持续合成，连击数累加，视觉效果逐级增强
步骤4a：5秒内未合成 → 连击中断 → 显示连击结算 → 恢复正常
步骤4b：达到10连击 → 进入合成狂热模式 → 金色花瓣飘落 → 15秒或连击中断后结束

5. 数值设计

| 参数 | 数值 |
|------|------|
| 连击时间窗口 | 5秒 |
| 狂热模式持续时间 | 15秒 |
| 合成溢出概率（狂热期间） | 10% |
| 经验加成范围 | +10% ~ +100% |
| 连击计时器位置 | 屏幕右上角（不遮挡棋盘） |

• 经验值说明：
  ◦ 每次基础合成获得经验 = 合成产物等级 × 10 经验
  ◦ 连击加成为额外百分比加成（如3连击：产物3级 → 基础30经验 → +20%加成 → 实际36经验）
  ◦ 经验用于玩家升级，升级奖励为全额恢复体力 + 少量钻石（与初版策划案一致）

6. UI设计要点

• 连击计时条：屏幕右上角弧形倒计时条（花瓣形状），颜色随连击数从淡金变深金变红
• 连击数字：每次合成后在合成位置上方弹出连击数字（"×2!"、"×3!"），字体大小随连击数递增
• 合成狂热模式：棋盘背景叠加金色半透明滤镜 + 花瓣粒子（从上方飘落）
• 连击结算浮窗：屏幕中央圆角卡片，显示连击数 + 额外经验，1.5秒渐隐
• 所有连击视觉效果不遮挡棋盘操作区域

7. 与现有系统的关联

• 合成事件：需要在 MergeManager 的合成成功回调中触发连击计数
• 经验系统：连击经验加成需要与玩家升级系统对接
• 日结算：连击记录需要纳入每日打烊结算展示
• 合成溢出：需要调用 BoardManager 的空格检查和物品放置逻辑

8. 开发量预估

• 连击计数与计时逻辑（0.5天）
• 连击UI效果（计时条、数字弹出、狂热模式视觉）（0.5天）
• 总计：1天

十、优化方案七：智能合成引导线（优先级 P2）

1. 功能概述

玩家拖拽物品时，除了高亮可合成目标格子外，还通过虚线箭头显示"推荐合成路径"——根据当前客人订单需求，智能推荐优先合成方向。

2. 需求来源

• 业界无类似设计，属于原创创新
• 解决痛点：新手玩家面对满棋盘物品时不知道该先合成什么、合成完给谁
• 降低决策负担，让"下一步该做什么"更清晰

3. 详细玩法描述

（1）拖拽时高亮增强（基础功能，在现有高亮基础上增强）
• 现有功能：拖拽物品时，棋盘上可合成的同类物品高亮
• 增强内容：可合成目标格子边框显示金色虚线高亮 + 目标物品轻微上下浮动

（2）推荐合成路径（核心创新）
• 当玩家拖拽一个物品时，系统根据当前客人订单需求，计算"最短路径"并显示推荐：
  ◦ 扫描当前所有客人的未满足需求
  ◦ 找到与当前拖拽物品同品类、同系的需求（如客人需要3级浪漫花束，当前拖拽的是1级浪漫花束）
  ◦ 计算从当前等级到目标等级的最短合成步骤
  ◦ 在棋盘上用虚线箭头串联显示推荐路径

（3）路径显示示例

场景：客人需要4级薰衣草花束，玩家拖拽1级粉玫瑰

棋盘上显示：
[1级粉玫瑰] --虚线箭头→ [另一个1级粉玫瑰] → 合成出2级百合
[2级百合] --虚线箭头→ [另一个2级百合（如果有）] → 合成出3级郁金香
最终指向客人头顶需求气泡中的目标物品

路径文字提示："合成后可满足 [客人名] 的订单！"

（4）显示规则
• 仅在拖拽开始后0.5秒显示推荐路径（避免短暂拖拽时频繁计算）
• 同时最多显示1条推荐路径（最高优先级的订单需求）
• 优先推荐距离最近（步骤最少）的可满足订单
• 如果当前物品与所有订单无关联，不显示推荐路径

（5）可关闭设置
• 设置页面增加开关："合成引导线"（默认开启）
• 老手玩家可关闭此功能，减少视觉干扰

4. 交互流程

步骤1：玩家长按某个物品进入拖拽状态
步骤2：拖拽开始后0.5秒，系统计算推荐路径
步骤3：推荐路径以虚线箭头形式显示在棋盘上
步骤4：路径起点为当前物品，终点指向客人需求气泡
步骤5：玩家将物品拖拽到推荐路径上的目标物品 → 合成
步骤6：合成完成后路径消失，如果新产物仍有推荐路径，下次拖拽时再次显示
步骤7：松手归位 → 路径消失

5. 数值设计

• 路径计算延迟：拖拽开始后0.5秒
• 最大推荐步骤数：3步（超过3步不显示，避免路径过长导致视觉混乱）
• 路径有效距离：棋盘内所有已解锁格子
• 路径刷新频率：拖拽过程中不实时刷新，仅在开始拖拽时计算一次

6. UI设计要点

• 推荐路径虚线：淡绿色虚线箭头（与合成提示的金色区分），线宽2px，虚线样式（6px线段+3px间隔）
• 箭头终点：小三角箭头指向目标物品
• 路径节点标注：路径经过的中间物品上显示小数字标注步骤（"①→②→③"）
• 最终指向客人气泡的箭头用半透明渐变效果（从棋盘区域延伸到花店场景区域）
• 路径文字提示：棋盘上方浮动小字"满足 [客人名] 的订单！"，淡绿色，半透明

7. 与现有系统的关联

• 订单数据：需要读取 CustomerManager 中当前所有客人的未满足需求
• 物品数据：需要读取 BoardManager 中当前棋盘所有物品的类型和等级
• 合成路径计算：基于 ItemConfig 的合成关系（getMergeResultId）反向推算
• 拖拽系统：在 BoardView 的拖拽事件中增加路径计算和显示逻辑

8. 开发量预估

• 推荐路径计算算法（0.5天）
• 路径虚线箭头渲染（0.5天）
• 与拖拽系统和订单系统集成（0.5天）
• 总计：1~2天

十一、优化方案八：季节花语主题棋盘（优先级 P3）

1. 功能概述

棋盘背景和格子样式随游戏内"季节"自动变化（春/夏/秋/冬循环），不同季节有特殊合成加成效果。增加长线新鲜感和策略规划维度。

2. 需求来源

• 业界无类似设计，属于原创创新
• 花店主题与四季变化有天然关联（不同季节有不同主打花卉）
• 增加长线游戏的新鲜感，避免视觉疲劳

3. 详细玩法描述

（1）季节循环规则
• 每个季节持续7天（现实时间），4个季节为一轮（28天一循环）
• 季节顺序：春天 → 夏天 → 秋天 → 冬天 → 春天 → ...
• 所有玩家共享同一季节（服务器时间统一）
• 季节切换时间：每周一0:00自动切换

（2）四季视觉变化

春天：
• 棋盘背景：浅粉色/浅绿色柔和渐变
• 格子边框：淡粉色描边
• 背景装饰：樱花花瓣轻微飘落粒子效果（稀疏，不影响操作）
• 季节图标：屏幕右上角显示樱花图标+"春"字

夏天：
• 棋盘背景：浅蓝色/浅绿色清新渐变
• 格子边框：淡绿色描边
• 背景装饰：绿叶光斑粒子效果
• 季节图标：绿叶图标+"夏"字

秋天：
• 棋盘背景：暖橙色/浅黄色温暖渐变
• 格子边框：淡橙色描边
• 背景装饰：红叶缓慢飘落粒子效果
• 季节图标：红叶图标+"秋"字

冬天：
• 棋盘背景：浅蓝灰色/白色清冷渐变
• 格子边框：淡蓝色描边
• 背景装饰：雪花稀疏飘落粒子效果
• 季节图标：雪花图标+"冬"字

（3）季节切换过渡动画
• 季节切换当天首次登录时播放过渡动画（约2秒）：
  ◦ 春→夏：樱花渐隐 → 绿叶渐显
  ◦ 夏→秋：绿叶变黄 → 红叶飘落
  ◦ 秋→冬：红叶渐隐 → 雪花渐显
  ◦ 冬→春：雪花消融 → 樱花绽放
• 过渡动画后弹出"季节更替"提示卡片，展示新季节名称+合成加成说明

（4）季节合成加成

| 季节 | 加成花系/饮品线 | 加成效果 | 主题说明 |
|------|---------------|---------|---------|
| 春天 | 日常花系 | 合成产出概率+1级（即有20%概率合成后直接产出高一级物品，等于跳一级） | 春天万物复苏，日常花卉繁盛 |
| 夏天 | 冷饮线 | 冷饮线生成器产出数量×2（每次点击产出2个） | 夏天冷饮畅销 |
| 秋天 | 浪漫花系 | 浪漫花系合成时经验加成×1.5 | 秋天浪漫告白季 |
| 冬天 | 甜品线 | 甜品线物品出售价格×2 | 冬天暖心甜品季 |

加成规则说明：
• 加成效果仅影响特定花系/饮品线，不影响全局
• 加成在季节期间持续生效，无需手动激活
• 棋盘左上角常驻显示当前季节加成提示图标（悬浮可查看详情）
• 加成效果叠加连击加成，但独立计算（不乘法叠加）

（5）季节限定目标（可选扩展，后续版本）
• 每个季节可设置季节限定订单：完成后获得季节限定装饰
• 季节结束时结算季节任务完成度

4. 交互流程

【季节切换】
步骤1：新一周到来，季节自动切换
步骤2：玩家当天首次登录 → 播放季节过渡动画（2秒）
步骤3：弹出"季节更替"卡片 → 显示新季节名称+加成说明
步骤4：点击关闭卡片 → 进入主页面，棋盘已切换为新季节视觉

【查看当前季节加成】
步骤1：点击棋盘左上角季节图标
步骤2：弹出浮层显示当前季节名称、剩余天数、加成效果详情

5. 数值设计

| 参数 | 数值 |
|------|------|
| 季节持续时间 | 7天/季 |
| 完整循环 | 28天 |
| 春天日常花系跳级概率 | 20% |
| 夏天冷饮线产出倍数 | ×2 |
| 秋天浪漫花系经验倍数 | ×1.5 |
| 冬天甜品线售价倍数 | ×2 |

• 数值影响分析：
  ◦ 每个加成仅影响27种商品中的部分（3~6种），对整体经济影响有限
  ◦ 加成更多是引导玩家在不同季节调整合成策略（"夏天多做冷饮"），增加策略感
  ◦ 不影响核心平衡，属于调味剂

6. UI设计要点

• 棋盘背景切换使用渐变过渡（不是硬切），色调变化应柔和自然
• 粒子效果（花瓣/绿叶/红叶/雪花）数量控制在屏幕上同时10~15个，避免性能问题
• 粒子落速缓慢（约5秒从上到下），营造静谧氛围
• 季节图标位于棋盘区域左上角，不遮挡格子，尺寸约30×30px
• 季节过渡动画不阻断操作（动画播放期间棋盘可正常操作）

7. 与现有系统的关联

• 时间系统：需要服务端统一季节时间，或基于客户端本地时间计算当前季节
• 合成系统：MergeManager 的合成逻辑中增加季节加成判断（跳级概率）
• 建筑系统：建筑产出逻辑中增加季节产出倍数判断
• 出售系统：物品出售价格逻辑中增加季节售价倍数判断
• 经验系统：经验计算中增加季节经验倍数

8. 开发量预估

• 季节系统核心逻辑（时间判定、加成计算）（0.5天）
• 棋盘背景切换与粒子效果（1天）
• 过渡动画与季节切换UI（0.5天）
• 各系统加成集成（合成/建筑/出售/经验）（1天）
• 总计：2~3天

十二、优化方案九：合成统计面板（优先级 P3）

1. 功能概述

记录并展示玩家的合成数据，包括今日合成次数、最高连击、合成出的最高级物品等。在日结算时展示，配合周/月合成达标奖励。

2. 需求来源

• 数据驱动的成就感和留存
• 配合连击系统（优化方案五）的数据记录
• 在已有的"打烊结算"环节增加内容展示

• 今日最高连击：本日最高连击记录
• 今日合成出的最高级物品：展示图标+名称
• 今日交付订单数：完成的客人订单数
• 今日消耗体力：总消耗体力值

历史数据（永久记录）：
• 累计合成次数
• 历史最高连击
• 已合成出的最高级物品
• 累计交付订单数
• 累计经营天数

（2）日结算展示增强
• 在现有"打烊结算"页面增加合成数据展示区域
• 布局：在营业额、服务客人数、好评率下方增加一栏"今日合成数据"
• 显示：今日合成次数 + 今日最高连击 + 今日最高合成物品图标

5. 数值设计

• 每周合成目标：100/300/500次（对应每天约14/43/71次合成，正常玩家每天合成50~80次，500次目标需要积极玩家）
• 每月合成目标：2000次（对应每天约67次，中度玩家可达成）
• 连击目标：8连/15连（需要熟练操作+合理规划，有一定挑战性但可达成）
• 奖励量级为辅助性质，不影响核心经济平衡

6. UI设计要点

• 日结算合成数据区域：与现有结算样式统一，增加分隔线+"今日合成数据"小标题
• 合成统计标签页：简洁的数据展示+进度条
• 周/月目标进度条：圆角进度条，已完成部分为金色，未完成为灰色
• 目标达成时进度条变为闪光效果 + "领取"按钮变为金色脉动

7. 与现有系统的关联

• 数据收集：在 MergeManager 的合成回调中累加计数
• 连击系统：复用优化方案五的连击数据
• 日结算：集成到现有的打烊结算页面
• 任务面板：增加"合成统计"标签页
• 数据持久化：合成统计数据纳入存档

8. 开发量预估

• 数据收集与存储逻辑（0.5天内含）
• 日结算页面增强 + 合成统计标签页UI（0.5天内含）
• 总计：0.5天

十四、预期效果分析

1. 对标差距消除

• 合成线可视化 + 仓库系统的实现，将消除我们与四季合合、浪漫餐厅在合成页面体验上的所有明显差距
• 可合成提示增强的完善，将补全策划案已提及但未完整落地的功能

2. 差异化竞争优势

实现全部9项优化后，合成页面将形成三个独有竞争维度：

（1）操作爽感维度：连击系统 + 双击快速合成
• 连续合成的即时反馈远超竞品，"合成狂热"模式提供独有的心流体验
• 双击快速合成提供竞品所没有的便捷操作方式

（2）情感体验维度：花语合成彩蛋
• 利用花店主题的天然文化属性，创造竞品无法复制的情感共鸣
• 花语文案对女性用户有极强的情感吸引力

（3）长线新鲜感维度：季节主题棋盘 + 合成统计
• 每周变化的棋盘视觉和合成策略，保持长线玩家的新鲜感
• 数据驱动的合成目标提供持续的短期目标感

3. 对核心指标的预期影响

| 指标 | 预期提升 | 主要贡献功能 |
|------|---------|------------|
| 新手留存率（次日留存） | +5~10% | 合成线可视化、可合成提示、智能引导线 |
| 7日留存率 | +3~5% | 连击系统、花语彩蛋、合成统计周目标 |
| 单次游戏时长 | +2~5分钟 | 连击系统（追求高连击）、仓库（更从容的操作空间） |
| 用户满意度 | 显著提升 | 仓库（解决痛点）、双击合成（效率提升）、花语彩蛋（情感惊喜） |
| 长线留存（30日） | +2~3% | 季节主题棋盘、合成统计月目标、花语图鉴 |

十五、结论

本策划案基于四季合合、浪漫餐厅、庄园合合、EverMerge四款业界标杆产品的合成页面深度调研，结合《花花妙屋》现有功能现状，提出9项具体优化方案。

核心结论：

1. 我们在合成页面的核心机制上已与业界持平（二合一、生成器、迷雾格、体力、订单），甚至在部分机制上优于竞品（自动锁定+手动确认交付、双品类合成）

2. 必须立刻补齐合成线可视化和仓库系统（P0），消除与业界的体验差距

3. 通过连击系统+花语彩蛋+双击合成（P1/P2），在操作爽感和情感体验上形成差异化竞争优势

4. 季节主题棋盘和合成统计（P3）作为长线内容储备，持续提供新鲜感和目标感

5. 全部9项功能预计10~14天完成开发，投入产出比极高

实现以上优化后，《花花妙屋》合成页面体验将全面超越四季合合和浪漫餐厅，在保持核心玩法对齐的基础上，通过"连击爽感 + 花语情感 + 操作便捷 + 季节策略"四个维度形成独有的竞争壁垒。
