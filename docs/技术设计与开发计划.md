# 《花花妙屋》PixiJS 微信小游戏技术设计与开发计划

## 一、核心技术决策

### 1.1 为什么选 PixiJS v7 而非 v8

**v8 在微信小游戏的已知严重问题：**

- `unsafe-eval` 限制：微信禁止 `new Function()`，v8 的 shader 编译依赖它
- VAO (Vertex Array Objects) 不被部分 Android 设备支持，v8 强制要求
- `OffscreenCanvas` 在微信真机部分实现但不完整，导致 PixiJS 走入错误代码分支
- v8 重构了 `DOMAdapter` 架构，适配层需要完全重写
- v8 移除了 Canvas2D 渲染器，无降级方案

**v7 的优势：**

- `pixi-miniprogram` 项目已验证 v7.3.2 在微信环境中稳定运行
- 社区方案成熟，`@pixi/unsafe-eval` 等模块久经考验
- 内置 Canvas2D 渲染器可作为降级方案
- 对本游戏规模（~110 个显示对象）性能完全满足

### 1.2 渲染方案：WebGL（业界主流）

**调研结论：微信小游戏 + PixiJS + WebGL 是业界验证可行的标准方案。**

核心依据：

- `pixi-miniprogram`（基于 PixiJS 7.3.2）使用完整 WebGL 管线在微信中稳定运行
- `finscn/weapp-adapter`（433 stars）支持 WebGL 引擎（PixiJS/ThreeJS/Babylon）
- `deepkolos/platformize` 提供了 pixi-wechat-game WebGL 示例
- PixiJS 7 的 Text 类内部使用 Canvas2D 渲染文字再上传为 WebGL 纹理，**只要 adapter 正确实现 `wx.createCanvas()` 离屏 canvas，文字渲染即可正常工作**
- 如遇极端兼容性问题，v7 可通过 `forceCanvas: true` 一键降级到 Canvas2D

**已知 WebGL 问题及业界解决方案：**

| 问题 | 解决方案 |
|------|---------|
| `HTMLImageElement` instanceof 失败 | 用 `wx.createImage().constructor` 直接赋值（非 extends） |
| `navigator.userAgent` 只读 | try-catch 包裹 |
| Android `OES_vertex_array_object` 兼容 | adapter 中提供 polyfill |
| `gl.texImage2D` 不接受微信对象 | 通过正确的原型链设置解决 instanceof 校验 |
| `OffscreenCanvas` 部分实现 | 设为 undefined 禁用 |
| `unsafe-eval` 限制 | 引入 `@pixi/unsafe-eval` |

### 1.3 技术栈确认

- 渲染引擎：PixiJS v7.3.x（WebGL 渲染器，Canvas2D 作为降级方案）
- 语言：TypeScript
- 构建：Vite (IIFE 格式打包)
- 动画：自建轻量 Tween 系统（避免引入额外依赖）
- 状态管理：自建事件总线 + 单例管理器模式
- 适配器：基于 finscn/weapp-adapter 模式的模块化 pixi-adapter

## 三、模块化 pixi-adapter 设计（跨平台 + 业界验证模式）

参考 `finscn/weapp-adapter`（433 stars）和掘金社区文章的模块化适配器模式，adapter 拆分为独立模块。**核心创新：增加 `platform.js` 平台抽象层，实现微信/抖音零成本切换。**

### 3.1 平台抽象层 platform.js（跨平台核心）

```javascript
// 自动检测当前平台，提供统一 API 接口
const isWechat = typeof wx !== 'undefined';
const isDouyin = typeof tt !== 'undefined';
const api = isWechat ? wx : isDouyin ? tt : null;

export default {
  // 渲染相关（微信和抖音完全一致）
  createCanvas: ()   => api.createCanvas(),
  createImage: ()    => api.createImage(),

  // 系统信息
  getSystemInfoSync: () => api.getSystemInfoSync(),

  // 存储（接口一致）
  getStorageSync: (k) => api.getStorageSync(k),
  setStorageSync: (k, v) => api.setStorageSync(k, v),

  // 网络（接口一致）
  request: (opts)       => api.request(opts),
  connectSocket: (opts) => api.connectSocket(opts),

  // 触摸事件（接口一致）
  onTouchStart:  (cb) => api.onTouchStart(cb),
  onTouchMove:   (cb) => api.onTouchMove(cb),
  onTouchEnd:    (cb) => api.onTouchEnd(cb),
  onTouchCancel: (cb) => api.onTouchCancel(cb),

  // 音频（接口一致）
  createInnerAudioContext: () => api.createInnerAudioContext(),

  // 平台标识
  name: isWechat ? 'wechat' : isDouyin ? 'douyin' : 'unknown',
  api,  // 原始平台对象，用于调用平台特有 API
};
```

**微信与抖音的 adapter 级 API 完全一致**（createCanvas、createImage、getStorageSync 等），差异只在前缀（wx/tt），所以通过这一层抽象，所有 adapter 模块不再写死 `wx.xxx`。

### 3.2 核心 adapter 模块设计

**canvas.js** — 全局 canvas 管理

```javascript
import api from './platform';
// 第一次 createCanvas() 返回主屏 canvas（微信和抖音行为一致）
const canvas = api.createCanvas();
export { canvas };
```

**element.js** — DOM 元素模拟（关键避坑点）

```javascript
import api from './platform';
export class Element {
  style = { cursor: null }
  appendChild() {}
  removeChild() {}
  addEventListener() {}
  removeEventListener() {}
}
// ⚠️ 业界已知坑：必须用 constructor 直接赋值，不能用 class extends
export const HTMLCanvasElement = api.createCanvas().constructor;
export const HTMLImageElement = api.createImage().constructor;
```

**document.js** — document 模拟

```javascript
import api from './platform';
import Image from './Image';
import { Element } from './element';
export default {
  body: new Element('body'),
  createElement(tag) {
    switch (tag) {
      case 'canvas': return api.createCanvas();  // 离屏 canvas
      case 'img': return api.createImage();
      default: return new Element();
    }
  },
  addEventListener(type, handle) { /* ... */ },
  removeEventListener(type, handle) { /* ... */ },
}
```

**index.js** — 统一入口（真机/模拟器环境分离处理）

```javascript
import api from './platform';
const { platform } = api.getSystemInfoSync();

if (platform === 'devtools') {
  // 模拟器：用 Object.defineProperties 挂载（window 上部分属性不可直接修改）
  Object.defineProperties(window, {
    Image: { value: Image },
    HTMLCanvasElement: { value: HTMLCanvasElement },
    HTMLImageElement: { value: HTMLImageElement },
    // ...
  });
} else {
  // 真机：直接挂载到 GameGlobal（微信和抖音都有 GameGlobal）
  GameGlobal.window = GameGlobal;
  GameGlobal.document = document;
  GameGlobal.Image = Image;
  GameGlobal.HTMLCanvasElement = HTMLCanvasElement;
  GameGlobal.HTMLImageElement = HTMLImageElement;
  // ...
}
```

### 3.3 避坑清单（基于业界实践 + 实测验证）

| 优先级 | 问题 | 解决方案 | 来源 |
|--------|------|---------|------|

| P1 | OffscreenCanvas 部分实现 | `GameGlobal.OffscreenCanvas = undefined` | pixi-miniprogram |
| P1 | 触摸事件坐标 | canvas.getBoundingClientRect 返回正确屏幕尺寸 | finscn/weapp-adapter |

## 四、跨平台架构设计（微信 → 抖音预留）

### 4.1 平台差异分析

| 对比项 | 微信小游戏 | 抖音小游戏 | adapter 是否需要改动 |
|--------|-----------|-----------|-------------------|
| 全局对象 | `GameGlobal` | `GameGlobal` | 不需要 |
| 创建 Canvas | `wx.createCanvas()` | `tt.createCanvas()` | **platform.js 已抽象** |
| 创建 Image | `wx.createImage()` | `tt.createImage()` | **platform.js 已抽象** |
| WebGL 支持 | 支持 | 支持 | 不需要 |
| Canvas 2D 支持 | 支持 | 支持 | 不需要 |
| 存储 API | `wx.get/setStorageSync` | `tt.get/setStorageSync` | **platform.js 已抽象** |
| 触摸事件 | `wx.onTouchStart` 等 | `tt.onTouchStart` 等 | **platform.js 已抽象** |
| 音频 | `wx.createInnerAudioContext` | `tt.createInnerAudioContext` | **platform.js 已抽象** |
| 定时器 | 全局可用 | 全局可用 | 不需要 |
| 入口文件 | `game.js` | `game.js` | 不需要 |
| 配置文件 | `game.json` | `game.json` | 格式基本一致 |
| 包体限制 | 主包 4MB / 总 20MB | 主包 4MB / 总 20MB | 不需要 |
| 登录 | `wx.login()` | `tt.login()` | 游戏层 PlatformManager 处理 |
| 支付 | `wx.requestPayment()` | `tt.pay()` | 游戏层 PlatformManager 处理 |
| 分享 | `wx.shareAppMessage()` | `tt.shareAppMessage()` | 游戏层 PlatformManager 处理 |
| 广告 | `wx.createRewardedVideoAd` | `tt.createRewardedVideoAd` | 游戏层 PlatformManager 处理 |
| 短视频能力 | 无 | `tt.createVideoPlayer()` | 抖音独有，游戏层处理 |

**结论：adapter 层通过 `platform.js` 已完全抽象，上抖音零改动。差异只在游戏业务层（登录、支付、分享、广告）。**

### 4.2 游戏业务层的跨平台设计

在 `src/managers/` 中预留一个 `PlatformManager.ts`：

```typescript
import platform from '../adapter/platform';

class PlatformManager {
  // 登录（接口不同，需分平台实现）
  login(): Promise<LoginResult> {
    if (platform.name === 'wechat') {
      return new Promise((resolve) => platform.api.login({ success: resolve }));
    } else if (platform.name === 'douyin') {
      return new Promise((resolve) => platform.api.login({ success: resolve }));
    }
  }

  // 分享（接口基本一致，参数略有差异）
  share(title: string, imageUrl: string) {
    platform.api.shareAppMessage({ title, imageUrl });
  }

  // 广告（接口一致）
  showRewardedAd(): Promise<boolean> {
    const ad = platform.api.createRewardedVideoAd({ adUnitId: this.getAdUnitId() });
    return ad.show().then(() => true).catch(() => false);
  }

  // 各平台的广告位 ID 不同
  private getAdUnitId(): string {
    return platform.name === 'wechat' ? 'wx_ad_id' : 'tt_ad_id';
  }
}
```

## 五、核心系统设计

### 5.1 棋盘系统（Board）

```
BoardManager
  - cells: Cell[30]              // 30 格状态数据
  - items: Map<cellIndex, Item>  // 物品数据

Cell {
  index, row, col,
  state: 'open' | 'fog' | 'peek' | 'key',
  presetContent: ItemId | null,
  keyPrice: number,
  unlockPriority: number
}

Item {
  id, category, line, level,
  isReserved, reservedBySlot
}
```

- 棋盘初始 4 格开放，其余 26 格按策划案预设布局
- 合成波及解锁：合成发生时检查相邻迷雾格，按等级决定解锁 1~2 格
- 跨格合成：拖拽到相邻可窥视格触发合成，结果留在已开放格

### 5.2 合成系统（Merge）

- 判定条件：同品类 + 同系 + 同等级 + 等级 < 最大等级
- 拖拽流程：pointerdown 记录起始格 -> pointermove 跟随 -> pointerup 判定目标格
- 合成结果：高一级物品出现在目标格，播放闪光动画
- 跨格合成：目标格为 peek 状态时特殊处理

### 5.3 客人系统（Customer）

- 刷新间隔 10~30 秒，最多同时 2 位客人
- 自动扫描棋盘匹配需求槽位，优先锁定最低等级物品
- 锁定物品可被合成消耗，消耗后自动重新扫描
- 全部满足后显示"完成"按钮，点击触发交付

### 5.4 存档系统（Save）

使用 `wx.setStorageSync` / `wx.getStorageSync`，定期自动保存：

- 棋盘状态（格子解锁状态 + 物品分布）
- 货币状态（金币、花愿、花露、钻石、体力）
- 建筑状态（CD 剩余、消耗型剩余次数）
- 客人状态（当前客人、锁定物品）
- 熟客好感度
- 离线时间戳（用于离线收益计算）
- 装修/换装解锁进度

## 六、渲染方案

**WebGL 渲染器（业界主流）+ Canvas2D 降级保底：**

```typescript
import '@pixi/unsafe-eval'; // 必须最先导入，解决 unsafe-eval 限制
import * as PIXI from 'pixi.js';

// 使用 PIXI.autoDetectRenderer 自动选择最佳渲染器
// 也可使用 Application 配置
const app = new PIXI.Application({
  view: canvas,            // adapter 提供的主屏 canvas
  width: GAME_WIDTH,
  height: GAME_HEIGHT,
  backgroundColor: 0xFFF5EE,
  // 默认使用 WebGL；若特定设备不支持，v7 会自动降级到 Canvas2D
  // 也可通过 forceCanvas: true 手动强制降级
});
```

**文字渲染策略（基于 pixi-miniprogram 经验）：**

- PixiJS 7 的 `PIXI.Text` 内部创建一个离屏 canvas 用 Canvas2D 绘制文字，再上传为 WebGL 纹理
- 只要 adapter 的 `document.createElement('canvas')` 正确返回 `wx.createCanvas()`，文字即可正常渲染
- 如需更高性能的文字渲染，可使用 `PIXI.BitmapText`（字体文件需部署到网络服务器）

**渲染规模评估（本游戏完全在 WebGL 舒适区内）：**

- 30 个格子 + 物品图标 ≈ 60 个显示对象
- 2 个客人 + 气泡 ≈ 20 个显示对象
- 店主 + 柜台 + UI ≈ 30 个显示对象
- 总计 ~110 个显示对象，WebGL 非常轻松
